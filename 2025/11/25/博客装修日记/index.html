博客有什么大的变动，我应该会第一时间写在这里。

现在

2026-06-08

今天把博客的评论区和阅读量统计从 LeanCloud 迁移到了 Waline。

一开始只是想找一个 LeanCloud 的替代品，后来发现 Waline 正好可以同时承担评论区、文章阅读量、全站访问量这些功能，于是就顺手把这一块重新整理了一遍。部署过程比想象中绕一点：Waline 部署在 Vercel 上，数据库用的是 Neon PostgreSQL，评论服务绑定到了 waline.awakefantasy.top 这个子域名。

迁移过程中还踩到了 Fluid 主题 1.9.9 的 Waline 加载问题。文章页明明有 #waline 容器，但 Fluid 的懒加载逻辑在某些情况下只等待“之后新增的元素”，导致 Waline 评论区不会真正初始化。查了一下上游 issue 和 PR，发现相关 PR 主要修的是 Waline CDN 和新版 DOM 选择器问题，并没有完整解决这个懒加载触发问题。最后在本地用 Hexo 渲染补丁绕过了这层问题，让评论区稳定显示。

旧数据也没有直接丢掉。我把 LeanCloud 导出的 Comment 和 Counter 数据转换后增量导入到了 Waline 的 PostgreSQL 表里，评论、文章阅读量、全站 PV/UV 都尽量保留下来了。中间还遇到了一点路径编码问题：LeanCloud 里保存的是中文解码路径，而浏览器里的 window.location.pathname 默认可能是 URL 编码后的路径，所以最后统一把文章阅读量路径规范化到中文解码路径，避免同一篇文章被统计成两条记录。

顺手还补了一些展示细节：现在首页每篇文章后面会显示对应阅读量；文章页顶部的元信息里，也会在“字数 / 阅读时间”后面显示阅读量；页脚则显示全站 PV/UV。整体上评论和统计这一块终于从“能用但分散”变成了“统一依赖 Waline”，以后维护起来应该会轻松很多。
